Running VASP with AiiDA

In HPC NERSC

Published

February 10, 2026

SSH

First one needs to get the sshproxy utility to connect to NERSC.

The instructions on how to install it are available on the NERSC website.

This generates an ssh key in your ssh folder (note that if your local username does not match the nersc username, you’ll have to use the -u flag)

Postgres

One needs to have Postgress installed.

$ brew install rabbitmq postgresql@14
$ verdi quicksetup

Start the process

verdi daemon start 2

Remove rabbitmq warnings

verdi config set warnings.rabbitmq_version False

Get the development aiida version (use eggg to tell package name)

pip install git+https://github.com/aiida-vasp/aiida-vasp.git@develop#egg=aiida-vasp

Add a computer

After setting

verdi computer setup --config perlmutter.yaml

Then running

$ verdi computer configure core.ssh perlmutter

Then this will ask some configureation. Just use the defaults, but point to your ssh file (be sure to set your username to the remote user).

Running VASP

We first have to configure a code. See the AiiDA-VASP documentation for setting up a code.

Running

$ verdi code list
Full label         Pk  Entry point
---------------  ----  -------------------
vasp@perlmutter     1  core.code.installed

Use `verdi code show IDENTIFIER` to see details for a code

Similar to pymatgen, we need AiiDA to raed the pseudopotentials.

I recommend using aiida-vasp eversion 4.1.0 (5.0.0 is still buggy)

Searching online for aiida-vasp shows the documentation found online for 3.1.0. One should go inside github and click the documentation link

verdi data vasp-potcar uploadfamily --path=/path/to/PBE --name=PBE.XX --description="..."

Make sure you install Python 3.11 and THEN scipy 1.13.X, since aiida-vasp requires cumtrapz (which was renamed in scipy>1.13).

Also, be sure to install ase==3.22.1 (or whichever version has ase.neb instead of ase.mep.neb).

All of this can also be done in

import os
from pathlib import Path
comp.set_workdir('/tmp/aiida_run/')
comp.configure()
vasp_path = check_output(['which', 'mock-vasp'], universal_newlines=True).strip()

vasp_code = orm.InstalledCode(comp, vasp_path, default_calc_job_plugin='vasp.vasp')
vasp_code.label ='mock-vasp'
vasp_code.store()
os.environ['MOCK_VASP_REG_BASE'] = str((Path() / 'mock_registry').absolute())
os.environ['MOCK_VASP_UPLOAD_PREFIX'] = 'band'

# Upload the POTCAR files
from aiida_vasp.data.potcar import PotcarData, PotcarFileData
from pathlib import Path

print(PotcarData.upload_potcar_family(str(Path('potcars').absolute()), "PBE.EXAMPLE", "PBE.EXAMPLE"))

# Setting up the silicon structure
from ase.build import bulk
si = bulk('Si', 'diamond', 5.4)
si_node = orm.StructureData(ase=si)

Handling errors

I hope this guide helps anyone in the handling of errors with AiiDA

process_node.exit_message: should tell you something like ‘No POTCAR found for full name Si in family PBE.54’

Be careful when handling potcars from NERSC or from Pymatgen,

Deleting a POTCAR family can be done through verdi group delete

Be sure to use only the Si potcars (i.e. don’t add Si_GW and the rest?)

Be careful how aiida-vasp names the potentials. “The name of the containing folder from which it was uploaded. This is used to specify a potential inside a family. Example: Zn_sv_GW” So since I had al my potentials stored in potpaw_PBE.54/ it was naming them as such.
